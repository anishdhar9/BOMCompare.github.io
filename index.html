How it works & assumptions
  • Match key: part number (Vault item “Number”), case-insensitive.
  • CAD BOM sources (upload one or, better, two together):
    • Vault multi-level BOM PDF (the web client's “Uses” report) — the full CAD structure including reference components; hierarchy from indentation; carries no quantities.
    • Inventor BOM export (.xlsx with Item / Part Number / QTY / BOM Structure) — the intended BOM: quantities and virtual components, but reference components are excluded from it.
    • Vault Excel paste (flat file listing) — full structure, no hierarchy or quantities; grouping is then inferred from the export's depth-first order plus the Item Master hierarchy.
  • Missing from Item Master: part numbers found in the CAD BOM but absent from the Item Master — the classic symptom of a component accidentally set to BOM Reference in Inventor.
  • Reference assemblies: when a whole assembly is missing, its child parts are grouped under it and collapsed — a reference assembly's entire BOM is expected to be absent, so only the assembly itself needs action.
  • Reference items (needs two CAD sources): components in the full CAD structure (PDF or Vault paste) that are absent from the Inventor BOM export — the exact list of what is currently flagged BOM Reference in the model, each marked whether it also made it into the Item Master. Review it to confirm every entry is intentional.
  • Quantity mismatches: rolled-up total quantity per part (own qty × parent assembly quantities, summed over all occurrences) compared between CAD and Item Master. Requires a source with a QTY column — that's the Inventor BOM export; the Vault PDF and flat paste carry no quantities.
  • In Item Master only: items whose number never appears in the CAD BOM — stale or manually added entries worth reviewing.
  • Item Master data quality (runs the moment the Item Master loads, no CAD BOM required): six checks on the file itself — Producer/Producer Number reflected in the top-level row's Description; the "END OF LINE" entry has the organization's fixed part number and a whole-number Row Order; the Quantity and Item Qty columns agree on every row (catches a manual edit to one that wasn't mirrored to the other); when the export includes an Entity Icon column, that it reads "Normal" everywhere; every row has a Title and Description (purchased/catalog parts, numbered "X-999-…", are only flagged when both are missing); and every non-assembly row has a Material (assemblies/weldments are excluded — they don't carry one). The QC report export includes a full copy of the Item Master with the flagged Title/Description/Material cells actually highlighted, not just listed.
  • Export file name: when the Item Master's top-level row carries the organization's SPN/PN project key (from its Producer / Producer Number columns, or parsed from the Description text), both the comparison export and the QC report include it in the file name.
  • Load from folder (Chrome/Edge only — a browser can't read a typed path, so this uses the browser's own folder picker): click, choose the PNxxxx project folder once, and the app finds the "Autodesk Vault…" PDF, the "EBOM_…" Item Master export, and (if present) a "PNxxxx_LLDBO" file inside it, loads them automatically, runs the comparisons, and writes the result workbook straight back into that folder — no download step. If a required file is missing or the folder has more than one candidate, that side is left for you to drop in manually; the normal upload boxes always work as a fallback.
  • Long-Lead Parts (LLDBO) — optional, checked once both it and the Item Master are loaded: long-lead direct-bought-out parts are released to procurement ahead of the normal BOM release to cover supplier lead times. This flags any long-lead part number that never made it into the Item Master (the early release didn't get captured, so it may quietly never get ordered through the normal channel) and any whose long-lead quantity (summed if the same part appears on more than one line) doesn't match the Item Master's rolled-up total. If the LLDBO document's own SPN/PN project key doesn't match the loaded Item Master's, a warning flags the likely mismatched files before anything else is reported. Rows not yet assigned a part number are counted separately and not treated as findings.
